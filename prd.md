## 1. 개요

### 1.1 배경 및 문제 정의

HR 담당자는 직원 채용·유지 과정에서 통근 부담이 이직률에 영향을 미친다는 것을 체감하지만, 이를 데이터로 정량화한 적이 없다.

**현재 한계:**
- 직원 거주지와 통근 시간이 시스템에 기록되지 않음
- 주거비 부담과 이직 의향의 상관관계를 분석할 데이터가 없음
- 거점오피스 입지나 주거 지원 정책 설계 시 데이터 없이 감에만 의존

이 서비스는 회사 위치를 기준으로 통근 시간과 아파트 실거래가를 공간적으로 결합하여, 통근 효율 대비 저평가된 '숨겨진 입지'를 발굴하고, AI 브리핑을 통해 개인 맞춤형 추천까지 제공한다.

### 1.2 핵심 가설

| 가설 | 검증 방법 |
|---|---|
| 통근 시간이 길수록 아파트 가격이 낮아지는 선형 패턴이 존재한다 | 선형 회귀 (통근시간 → 평당가) + R² 확인 |
| 회귀선 아래에 위치한 지역(Blue Zone)이 실거주 가성비가 높다 | 잔차 분석으로 under-valued 지역 식별 |
| 지하철 직결 여부가 직선거리보다 통근 시간에 더 큰 영향을 미친다 | 직선거리 vs 카카오맵 실제 통근시간 비교 |

## 2. 목표 및 성공 지표

### 2.1 제품 목표
- 특정 회사 위치 기준 통근 반경(30분/1시간)별 아파트 실거래가 히트맵 시각화
- 선형 회귀 기반 잔차 분석으로 **Blue Zone(저평가)** / **Red Zone(프리미엄)** 자동 분류
- LLM 브리핑: 예산·우선순위 입력 시 최적 동네 3곳 텍스트 추천
- HR 의사결정(거점오피스, 주거 지원 정책) 지원 리포트 출력

### 2.2 성공 지표 (KPI)

| 지표 | 목표값 | 측정 방법 |
|---|---|---|
| 지도 렌더링 시간 | 3초 이내 | 브라우저 로딩 타임 |
| 실거래가 데이터 최신성 | 최근 3개월 이내 | 국토부 API 갱신 주기 |
| 잔차 분석 모델 설명력 | R² ≥ 0.5 목표 | Scikit-learn 회귀 결과 |
| LLM 브리핑 응답 시간 | 10초 이내 | API 응답 측정 |
| Blue Zone 추천 만족도 | 사용자 체감 유용성 70%+ | HR 담당자 테스트 |

## 3. 사용자 및 사용 시나리오

### 3.1 주요 사용자 (User Personas)

| 사용자 | Pain Points | 주요 질문 |
|---|---|---|
| HR 담당자 | 직주근접 지원 정책의 실효성 부족, 거점오피스 후보지 근거 없음 | "통근 1시간+ 직원이 밀집한 지역은 어디? 거점오피스 놓으면 얼마나 줄어?" |
| 임직원 개인 | 예산 내 통근-주거비 트레이드오프 계산이 어려움 | "판교 40분 이내, 전세 3억 이하, 가성비 최고 동네 어디야?" |
| 기업 의사결정자 | 인재 유지를 위한 공간 전략 근거 데이터 부재 | "우리 직원 거주 분포를 보고 복지 예산을 어디에 써야 할까?" |

### 3.2 핵심 사용 시나리오

**시나리오 A — HR 거점오피스 후보지 탐색**

HR 담당자 A씨는 강남구 삼성동 본사 기준으로 통근 1시간 이상 직원이 많이 거주하는 지역을 파악하고 싶다. 서비스에 회사 주소를 입력하면 히트맵이 그려지고, Blue Zone 지역 중 수원, 용인 외곽이 거점오피스 후보로 추천된다. LLM이 "이 지역은 통근 시간 대비 평당가가 주변 평균보다 18% 저렴하며, 직원 거주 밀도도 높습니다"라고 브리핑한다.

**시나리오 B — 임직원 개인 이사 지역 탐색**

판교테크노밸리 재직 중인 B씨가 예산 5억, 통근 40분 이내 조건을 입력한다. LLM이 Blue Zone 중 조건에 맞는 동네 3곳(광주시 오포읍, 용인시 수지구, 성남시 분당구 일부)을 이유와 함께 브리핑하고, 지도에서 해당 지역이 하이라이트된다.

### 4.2 F-04 잔차 분석 상세 스펙

**핵심 로직:** 통근시간(X) → 평당가(Y) 선형 회귀 후, 실제값과 예측값의 차이(잔차)로 지역 분류

**분류 기준:**
- **Blue Zone** (Under-valued): 잔차 < -1σ → 통근 시간을 감안했을 때 기대보다 저렴한 지역
- **Red Zone** (Premium): 잔차 > +1σ → 학군·인프라 등 추가 프리미엄이 붙은 지역
- **Gray Zone** (Neutral): |잔차| < 1σ → 시장 적정가 수준

```python
from sklearn.linear_model import LinearRegression

model = LinearRegression().fit(X_commute, y_price)
residuals = y_price - model.predict(X_commute)
df['zone'] = pd.cut(residuals, bins=[-inf, -sigma, sigma, inf],
                    labels=['Blue', 'Gray', 'Red'])
```

### 4.3 F-05 LLM 브리핑 상세 스펙

**입력 파라미터:**
- 예산 (예: 전세 5억 이하)
- 통근 시간 상한 (예: 40분 이내)
- 우선순위 자유 입력 (예: "학군보다 통근이 중요", "조용한 동네 선호")

**프롬프트 설계 원칙:**
- 분석된 Blue Zone 데이터를 컨텍스트로 주입 (RAG 방식)
- 추천 동네 3곳 + 각 이유 + 주의사항을 구조화된 텍스트로 출력
- 할루시네이션 방지: 실거래가 데이터에 없는 지역은 추천하지 않도록 제약

**사용 API:** OpenAI API (GPT-4o 권장)
